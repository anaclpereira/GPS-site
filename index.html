Conceitos do mundo da Informática que você precisa saber

Git

O Git é um sistema de controle de versão de código fonte. Ele é usado por programadores para gerenciar e controlar mudanças em seus projetos de software. Em outras palavras, o Git é uma ferramenta que ajuda a manter o controle das diferentes versões do código fonte de um projeto e a trabalhar em equipe em um mesmo projeto, evitando conflitos.

GitFlow

Já o GitFlow é um modelo de fluxo de trabalho, ou seja, uma forma organizada e padronizada de usar o Git. Ele se baseia em dois ramos principais: o "develop" (desenvolvimento) e o "master" (produção). O "develop" é onde as mudanças são feitas e testadas antes de serem mescladas com o "master", que contém a versão mais atualizada e estável do projeto. Além desses ramos principais, o GitFlow também define outros ramos secundários, como "feature" (funcionalidade), "release" (versão de produção) e "hotfix" (correção urgente de bug).

Docker

Docker é uma plataforma de software que permite criar, testar e implantar aplicativos em contêineres. Um contêiner é uma unidade de software que inclui tudo o que um aplicativo precisa para ser executado, como código fonte, bibliotecas e dependências. Isso significa que o Docker permite que os desenvolvedores criem e implantem aplicativos de forma rápida e consistente, independentemente do ambiente de execução. Por exemplo, um desenvolvedor pode usar o Docker para criar um contêiner que inclui um aplicativo web, um servidor web, um banco de dados e todas as dependências necessárias. Esse contêiner pode ser facilmente implantado em diferentes sistemas operacionais, ambientes de desenvolvimento ou infraestruturas de nuvem, sem a necessidade de instalar ou configurar manualmente todas as dependências e componentes.

CI/CD

CI/CD é uma prática de desenvolvimento de software que se baseia em duas etapas principais: Integração Contínua (CI) e Entrega Contínua/Implantação Contínua (CD). A Integração Contínua é um processo de automação que permite que desenvolvedores integrem o código-fonte em um repositório centralizado várias vezes ao dia, em vez de fazer uma grande integração no final do ciclo de desenvolvimento. Já a Entrega/Implantação Contínua é um processo de automação que permite que as alterações de código sejam testadas, empacotadas e entregues automaticamente em ambientes de produção ou em um ambiente de preparação para produção. Em outras palavras, CI/CD é um conjunto de práticas que ajudam as equipes de desenvolvimento a criar, testar e implantar software de forma rápida e consistente. Com o uso de ferramentas de automação, é possível integrar e testar continuamente o código-fonte, garantindo que as alterações sejam de alta qualidade e funcionem corretamente em diferentes ambientes. O objetivo final do CI/CD é permitir que as equipes de desenvolvimento entreguem software com mais rapidez e eficiência, reduzindo o tempo entre o desenvolvimento e a implantação em produção.

Computação em Nuvem

Computação em Nuvem é uma tecnologia que permite o acesso sob demanda a recursos de computação, como servidores, armazenamento, banco de dados, redes e software, por meio da internet. A computação em nuvem oferece vários benefícios, como a possibilidade de escalar recursos de forma elástica, de acordo com as necessidades do usuário, o que permite a economia de custos, e o acesso a recursos que normalmente seriam muito caros ou complexos de gerenciar. O modelo IaaS oferece recursos de infraestrutura básicos, como servidores e armazenamento, que os usuários podem configurar e gerenciar de acordo com suas necessidades. O modelo PaaS oferece uma plataforma completa de desenvolvimento e implantação de aplicativos, incluindo ferramentas de desenvolvimento, banco de dados e sistemas de gerenciamento.
